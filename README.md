# Banking-Machine-Learning-Project
This project explores a banking dataset to understand customer financial behavior and build machine learning models for predictive analysis.

## Project Overview
The goal of this project was to predict whether a loan applicant is likely to default based on their financial history and transaction habits. This project demonstrates the full Data Science lifecycle: from raw data cleaning to model deployment.

# Objectives

- Understand and explore structured banking data
- Apply regression techniques to predict loan amounts
- Evaluate model performance using statistical metrics
- Detect patterns in financial behavior
- Compare multiple machine learning models
- Build a complete end-to-end ML workflow
   
## Key Insights & Journey
This project covers fundamental to intermediate ML topics and part of my learning journey in Data Analytics and Machine Learning through hands-on practice using real world datasets.

# Diagnostics & Validation
Linearity & residual analysis
Multicollinearity (Variance Inflation Factor - VIF)
Outlier detection (Cook’s Distance)
# Feature Engineering
Dummy variable encoding
Feature selection (correlation, variance threshold)
# Model Improvement
Regularization (Ridge & Lasso)
Scaling (Standardization)
Overfitting control
# Advanced Models
Decision Trees

# Deployment

# Key Insights
- Customer financial behavior can be used to predict loan-related outcomes
- Multiple features improve prediction accuracy compared to single-variable models
- Regularization helps reduce overfitting and improves model generalization
- Tree-based models capture non-linear relationships better than linear models
